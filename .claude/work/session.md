### Design decision: enrichment priority order

Collection names are private (not in code). Priority order for Claude enrichment is stored in `config/collections.json` as an `enrichment_order` integer field (null = not in scope).

```json
"My Priority Collection": {
  "slug": "my-priority",
  "numeric_id": "...",
  "group": "Hustling",
  "extract": true,
  "enrichment_order": 1
}
```

`pilot_collections_by_enrichment_priority()` in `pipeline/collections.py` returns entries where `enrichment_order` is not null, sorted ascending. User sets these values manually in `config/collections.json`.

## Locked Technical Decisions

| Concern | Decision |
|---|---|
| Transcript engine | yt-dlp + faster-whisper base int8 |
| OCR engine | RapidOCR (rapidocr-onnxruntime==1.4.4) |

| enrichment priority order | `enrichment_order: int` field in collections.json — no names in code |

| CDN domain | instagram.fblr22-*.fna.fbcdn.net |
| Cookie format for yt-dlp | JSON → Netscape at runtime → tmp/cookies.txt |
